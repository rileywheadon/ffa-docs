Non-Stationary FFA

In this vignette, we will conduct a flood frequency analysis (FFA) on a non-stationary dataset. The ffaframework package is capable of handling three types of non-stationarity:

  1. A linear trend in the mean (location parameter) of the probability model.
  2. A linear trend in the variance (scale parameter) of the probability model.
  3. A linear trend in both the mean and variance of the probability model.

If you have not yet read the vignette on stationary FFA, we strongly recommend reading it first.

Setup

We begin by loading the example dataset Application_3.1.csv, which has two columns:

  • max: the AMS measurements
  • year: the corresponding observation year
library(ffaframework)
csv_path <- system.file("extdata", "Application_3.1.csv", package = "ffaframework")
df <- read.csv(csv_path)
df <- subset(df, !is.na(max))

The trend Object

The ffaframework package uses an R list called the trend to define the non-stationary signature in the data. The trend list contains two boolean items: location and scale. If location = TRUE, there is a trend in the mean, and if scale = TRUE, there is a trend in the variance. This vignette will assume that we have already identified a trend in the mean (and no trend in the variance) in Application_3.1.csv. For more information about how to identify trends in your own datasets, see the vignette on trend identification.

trend <- list(location = TRUE, scale = FALSE)

Distribution Selection

Distribution selection for non-stationary models also uses the method of L-moments, with one additional step. Before we can call our chosen selection function, we need to decompose the data to remove the non-stationary trend. We do this with the ams_decomposition function, which takes a vector of streamflow observations, the corresponding vector of years, and the trend object. The ams_decomposition function will return a decomposed vector of streamflow observations with the trend removed, which we can then pass to our selection function of choice.

data_decomposed <- ams_decomposition(df$max, df$year, trend)

selection <- select_zstatistic(data_decomposed)

print(selection$recommendation)
#> [1] "GNO"

plot_lmom_diagram(selection)

The Z-statistic selection method recommends the generalized normal (GNO) distribution.

Parameter Estimation

To estimate the parameters of the generalized normal distribution with a trend in the mean, we need to use maximum likelihood estimation, since the method of L-moments cannot handle non-stationarity. The fit_maximum_likelihood function implements maximum likelihood estimation for both stationary and non-stationary distributions. It has two required arguments:

  • data: The annual maximum streamflow observations.
  • model: A three-letter code corresponding to a probability distribution (ex. "GNO").

Since our dataset has a non-stationary trend, we need to provide two additional arguments:

  • years: The corresponding vector of years for the observations in data.
  • trend: The non-stationary trend object described above.

Unlike the fit_lmom methods, the fit_maximum_likelihood estimation method returns a list with two objects. The params item contains the fitted parameters while the mll item contains the maximum log-likelihood.

fit <- fit_maximum_likelihood(
    df$max,
    "GNO",
    years = df$year,
    trend = trend
)

print(fit$params)
#> [1] 224.0496619 -35.5153685  54.6324886  -0.3689085

print(fit$mll)
#> [1] -590.7329

Note that fit$params has four elements because of the non-stationary trend. In order, the parameters are \((\mu_{0}, \mu_{1}, \sigma, \kappa)\). At time slice \(t\), the fitted probability distribution is \(\text{GNO}(\mu_{0} + \mu_{1}t, \sigma , \kappa)\).

Uncertainty Quantification

Uncertainty quantification is also essential for non-stationary probability distributions. In addition to the uncertainty_bootstrap quantification method discussed in the Stationary FFA vignette, the FFA framework also implements the regula-falsi profile likelihood method which is used alongside maximum likelihood estimation. The uncertainty_rfpl method has two required arguments:

  • data: The annual maximum streamflow observations.
  • model: A three-letter code corresponding to a probability distribution (ex. "GNO").

Since our dataset has a non-stationary trend, we need to provide three additional arguments:

  • years: The corresponding vector of years for the observations in data.
  • trend: The non-stationary trend object described above.
  • slice: The year used to compute return levels (since the distribution changes with time).
uncertainty <- uncertainty_rfpl(
    df$max,
    "GNO",
    years = df$year,
    trend = trend,
    slice = 2025
)

plot_uncertainty(uncertainty)

Under non-stationarity, the meaning of a return period becomes more nuanced. Since the probability distribution is changing with time, a return period cannot be interpreted as the expected time to observe a flood with a given severity. Instead, we use the phrase effective return period to indicate that the reported return levels are based on the probability distribution for a specific year.
